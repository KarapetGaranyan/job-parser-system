# Инструкция по запуску и использованию Job Parser System (без Docker)

1. Клонируйте проект с GitHub:

    git clone https://github.com/ВАШ_ПОЛЬЗОВАТЕЛЬ/job-parser-system.git
    cd job-parser-system

2. Создайте виртуальное окружение:

    python -m venv .venv

3. Активируйте виртуальное окружение:

    # Для Windows:
    .venv\Scripts\activate
    # Для Linux/Mac:
    source .venv/bin/activate

4. Установите зависимости:

    # Для backend:
    cd backend
    pip install -r requirements.txt

    # Для frontend:
    cd ../frontend
    pip install -r requirements.txt

5. Запустите backend:

    cd backend
    python app.py

    # Backend будет работать на http://localhost:5000

6. Запустите frontend (в новом окне терминала):

    cd frontend
    python app.py

    # Frontend будет работать на http://localhost:8000

7. Откройте браузер и перейдите на:

    http://localhost:8000

8. Поиск вакансий:
    - Введите параметры (сайт, вакансия, количество, сортировка)
    - Нажмите "Поиск"
    - Дождитесь завершения (индикатор загрузки)
    - Просмотрите результаты, экспортируйте в CSV при необходимости

9. Просмотр вакансий:
    - В браузере (JSON): http://localhost:5000/api/vacancies
    - В браузере (текст): http://localhost:5000/vacancies_text

10. Экспорт вакансий в CSV (Excel):
    - Скачать файл: http://localhost:5000/export_csv
    - Открыть файл vacancies.csv в Excel или Google Таблицах

11. Использование CLI-скрипта:
    - Перейдите в папку backend
    - Запустите: python cli_search.py
    - Следуйте инструкциям (выбор сайта, вакансия, количество, сортировка)
    - Новые вакансии сохраняются в vacancies_cli.json

---

**Важные замечания:**
- Если поиск не работает (ошибка 404) — проверьте, что в JS/HTML указаны полные адреса backend (http://127.0.0.1:5000)
- Для ускорения поиска ограничивайте количество страниц и вакансий
- Для корректного экспорта в Excel используйте CSV с BOM
- Для запуска через Docker используйте docker-compose.yml

---

Если появятся ошибки — скопируйте их и отправьте разработчику для помощи!

Теперь вы можете просматривать все вакансии из базы в виде обычного читаемого текста по адресу:
http://localhost:5000/vacancies_text
