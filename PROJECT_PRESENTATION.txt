# Описание проекта Job Parser System

## Кратко о проекте
Job Parser System — это система для автоматического поиска и сбора вакансий с крупнейших платформ HH.ru и SuperJob.ru с возможностью просмотра, фильтрации и сохранения результатов в базе данных.

---

## Использованные технологии
- **Python** — основной язык программирования
- **Flask** — веб-фреймворк для backend и frontend
- **requests** — для HTTP-запросов и работы с API
- **BeautifulSoup** — для парсинга HTML-страниц (HH.ru)
- **SQLAlchemy** — для работы с базой данных (ORM)
- **SQLite** — простая встроенная база данных для хранения вакансий
- **Selenium** (ранее, для обхода защиты сайтов, сейчас не используется)
- **SuperJob API** — официальный API для получения вакансий
- **HTML, CSS, JS** — для пользовательского интерфейса

---

## Архитектура
- **Backend** (Flask):
    - Эндпоинты для поиска вакансий, просмотра, выгрузки
    - Парсинг HH.ru через requests + BeautifulSoup
    - Получение вакансий с SuperJob.ru через официальный API
    - Сохранение всех вакансий в базу данных SQLite
    - Эндпоинты для просмотра вакансий в JSON и текстовом виде
- **Frontend** (Flask + HTML):
    - Форма поиска вакансий
    - Вывод результатов с пометкой источника (HH.ru или SuperJob)
    - Отправка запросов к backend через JS

---

## Основные методы и подходы
- **Парсинг HTML**: BeautifulSoup для HH.ru (поиск по CSS-классам и data-атрибутам)
- **Работа с API**: requests для SuperJob API (официальный способ, не блокируется)
- **Сохранение данных**: ORM SQLAlchemy, чтобы избежать дублирования и упростить работу с БД
- **REST API**: для интеграции с фронтендом и внешними сервисами
- **Обработка ошибок**: try/except, таймауты, логирование ошибок парсинга

---

## Особенности и плюсы
- Поиск сразу на двух платформах (HH.ru и SuperJob.ru)
- Использование официального API SuperJob для надёжности
- Хранение всех данных в базе (SQLite)
- Возможность просмотра вакансий в браузере (JSON и текст)
- Лёгкая доработка под другие платформы
- Устойчивость к ошибкам и блокировкам

---

## Минусы и ограничения
- Парсинг HH.ru может ломаться при изменении структуры сайта
- Нет сложной фильтрации и аналитики (можно добавить)
- Для больших объёмов данных лучше использовать PostgreSQL/MySQL
- Нет авторизации и разграничения прав (можно добавить)
